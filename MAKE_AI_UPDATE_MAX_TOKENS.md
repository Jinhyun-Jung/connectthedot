# Make AI Max Tokens 업데이트 가이드

## 📋 변경 사항

노드 본문 정보(단어, 뜻, 예문, 링크)를 포함하도록 프롬프트가 업데이트되어, 응답 길이가 길어질 수 있습니다.

**필수 업데이트**: Make AI 시나리오의 OpenAI 모듈에서 **Max tokens**를 `2000` → `4000`으로 변경해야 합니다.

---

## ✅ 업데이트 방법

### 1단계: Make AI 시나리오 열기

1. Make AI 대시보드 접속
2. 해당 시나리오 클릭하여 편집 모드로 진입

### 2단계: OpenAI 모듈 찾기

1. 시나리오에서 **OpenAI 모듈** (두 번째 모듈) 클릭
2. 모듈 설정 화면이 열림

### 3단계: Max Tokens 설정 변경

1. **Max tokens** 필드 찾기
   - 현재 값: `2000`
2. **값 변경**: `4000` 입력
   - 이유: 노드 본문 정보(JSON 형식) 포함으로 응답이 길어질 수 있음
3. **저장**: "OK" 또는 "Save" 버튼 클릭

### 4단계: 시나리오 저장

1. 시나리오 편집 화면에서 **"Save"** 버튼 클릭
2. 변경사항 저장 완료

---

## 🔍 확인 방법

### 테스트 실행

1. 시나리오에서 **"Run once"** 버튼 클릭
2. OpenAI 모듈 클릭하여 설정 확인:
   - **Max tokens**: `4000` ✅
   - **Temperature**: `0.3`
   - **Model**: `gpt-4o`
3. 실행 후 응답이 정상적으로 반환되는지 확인

---

## ⚠️ 중요 사항

### Max Tokens가 부족한 경우

- **증상**: 응답이 중간에 잘림
- **원인**: Max tokens가 부족하여 전체 응답을 생성하지 못함
- **해결**: Max tokens를 더 늘리기 (예: 4000 → 6000)

### Max Tokens가 너무 큰 경우

- **문제**: 비용 증가 (토큰 사용량 증가)
- **권장**: 필요한 만큼만 설정 (4000 권장)

## 📊 권장 설정

| 설정 항목 | 권장 값 | 설명 |
|---------|--------|------|

| **Temperature** | `0.3` | 일관된 결과를 위해 낮은 값 사용 |
| **Max tokens** | `4000` | 노드 본문 정보 포함으로 증가 |
| **Top P** | (비워둠) | 기본값 사용 |

| **Presence penalty** | (비워둠) | 기본값 사용 |

### 클라이언트 코드 변경

- 클라이언트 코드(`js/ai-mermaid-converter.js`)에서도 `max_tokens: 4000`으로 설정되어 있습니다

### 응답 형식

- **Mermaid 코드**: 마인드맵 구조
- **JSON 본문 정보**: 노드별 상세 정보 (단어, 뜻, 예문, 링크)

## 📞 문제 해결

### 문제 1: 응답이 중간에 잘림

**해결 방법:**
- Max tokens를 더 늘리기 (4000 → 6000)
- 또는 프롬프트를 더 간결하게 수정

### 문제 2: Max tokens 필드를 찾을 수 없음

**해결 방법:**
- "Show advanced settings" 또는 "More options" 클릭
- 또는 "Parameters" 섹션 확장

### 문제 3: 설정 변경 후에도 응답이 짧음

**해결 방법:**
- 시나리오를 저장했는지 확인
- 시나리오가 "ON" 상태인지 확인
- "Run once"로 다시 테스트

## 💰 비용 고려사항

### 토큰 사용량

- **이전**: Max tokens 2000
- **변경 후**: Max tokens 4000
- **증가율**: 2배

### 비용 영향

- OpenAI API 비용은 토큰 사용량에 비례
- Max tokens를 늘리면 최대 비용이 증가할 수 있음
- 실제 사용량은 응답 길이에 따라 다름

### 권장사항

- 필요에 따라 Max tokens 조정
- 일반적으로 4000으로 충분함
- 응답이 자주 잘리는 경우에만 더 늘리기
