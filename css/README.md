# CSS 모듈 구조 가이드

## 📁 파일 구조

```
css/
├── 📄 styles.css         ← 메인 CSS 파일 (모든 모듈을 import)
├── 📄 variables.css      ← CSS 커스텀 프로퍼티 (변수)
├── 📄 base.css          ← CSS 리셋 및 기본 스타일
├── 📄 layout.css        ← 레이아웃 관련 (별 배경, 캔버스, 패널)
├── 📄 components.css    ← UI 컴포넌트 (노드, 버튼, 캘린더)
├── 📄 editor.css        ← 에디터 관련 (에디터, 포맷팅, 색상)
├── 📄 features.css      ← 특수 기능 (폴더, 드래그드롭, 인증, 스프레드시트)
├── 📄 responsive.css    ← 반응형 디자인 및 미디어 쿼리
└── 📄 README.md         ← 이 문서
```

## 🎯 각 파일의 역할

### 1. **variables.css** - CSS 변수
- 색상 팔레트
- 간격 시스템
- 폰트 설정
- 트랜지션 설정
- Z-index 스케일

### 2. **base.css** - 기본 스타일
- CSS 리셋
- 기본 HTML 요소 스타일
- 유틸리티 클래스
- 스크롤바 설정
- 성능 최적화 설정

### 3. **layout.css** - 레이아웃
- 별 배경 애니메이션
- 캔버스 시스템
- 패널 시스템
- 기본 레이아웃 구조

### 4. **components.css** - UI 컴포넌트
- 노드 시스템
- 버튼 시스템 (FAB, 일반 버튼)
- 캘린더 시스템
- 감정 버튼

### 5. **editor.css** - 에디터 기능
- 에디터 인터페이스
- 텍스트 포맷팅 도구
- 색상 선택 시스템
- 입력 필드 스타일

### 6. **features.css** - 특수 기능
- 폴더 시스템
- 드래그 앤 드롭
- 컨텍스트 메뉴
- 인증 시스템
- 스프레드시트 기능

### 7. **responsive.css** - 반응형 디자인
- 모바일 스타일
- 태블릿 스타일
- 데스크톱 스타일
- 접근성 설정
- 인쇄 스타일

## 🚀 장점

### ✅ **유지보수성**
- 기능별로 파일이 분리되어 있어 코드 찾기 쉬움
- 특정 기능 수정 시 해당 파일만 편집하면 됨
- 코드 충돌 위험 감소

### ✅ **성능 최적화**
- 필요한 모듈만 선택적으로 로딩 가능
- CSS 파일 크기 관리 용이
- 브라우저 캐싱 효율성 향상

### ✅ **팀 개발**
- 여러 개발자가 동시에 다른 모듈 작업 가능
- Git 병합 충돌 최소화
- 코드 리뷰 효율성 향상

### ✅ **디버깅**
- 문제 발생 시 해당 모듈만 확인하면 됨
- 개발자 도구에서 스타일 추적 용이
- 테스트 시 특정 기능만 격리 가능

## 🔧 사용 방법

### 기본 사용
HTML에서 메인 CSS 파일만 로드하면 모든 스타일이 적용됩니다:

```html
<link rel="stylesheet" href="css/styles.css">
```

### 선택적 로딩 (성능 최적화)
특정 기능만 필요한 경우 개별 모듈만 로딩할 수 있습니다:

```html
<!-- 기본 필수 파일들 -->
<link rel="stylesheet" href="css/variables.css">
<link rel="stylesheet" href="css/base.css">
<link rel="stylesheet" href="css/layout.css">

<!-- 필요한 기능만 선택적으로 로딩 -->
<link rel="stylesheet" href="css/components.css">
<link rel="stylesheet" href="css/responsive.css">
```

## 📝 수정 가이드

### 새로운 기능 추가 시
1. 해당 기능에 맞는 CSS 파일을 선택
2. 새로운 기능이 기존 카테고리에 맞지 않으면 새 파일 생성
3. `styles.css`에 새 파일 import 추가

### 스타일 수정 시
1. 수정하려는 요소가 어느 파일에 있는지 확인
2. 해당 파일에서 수정
3. CSS 변수 수정이 필요하면 `variables.css` 먼저 확인

### 반응형 수정 시
- `responsive.css` 파일에서 미디어 쿼리 수정
- 기본 스타일은 해당 컴포넌트 파일에서 수정

## ⚠️ 주의사항

- **파일 순서**: `styles.css`의 import 순서는 의존성을 고려하여 설정됨
- **변수 사용**: 새로운 색상이나 크기 값 추가 시 `variables.css`에 변수로 정의 후 사용
- **코드 중복**: 같은 스타일이 여러 파일에 중복되지 않도록 주의
- **네이밍**: CSS 클래스명은 해당 모듈의 역할을 명확히 나타내도록 작성
